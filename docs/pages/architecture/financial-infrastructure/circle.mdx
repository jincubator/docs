# Reference Architecture: Circle

This wrapper keeps Circle architecture in the Architecture section while preserving the canonical research dossier.

Circle is a reference architecture, not a Jincubator-built system. It is useful as a financial-infrastructure architecture lens across public APIs, wallets, contracts, CCTP, stablecoins, platform controls, and operational boundaries.

## Architecture Lens

- API platform surfaces for wallets, contracts, CCTP, gateway, minting, and payment products.
- Stablecoin control surfaces including mint, burn, pause, blacklist, permit, and upgradeability.
- CCTP burn, mint, message, attestation, finality, and replay-protection boundaries.
- Evidence discipline around public docs, OpenAPI schemas, repositories, and labeled assumptions.
